# 咖啡订购Agent 设计文档

## 项目概述
这是一个面向个人使用的咖啡订购模拟Agent，支持完全自然语言交互，不需要对接真实的外卖平台API，内置模拟咖啡店数据，可模拟完整的点单到配送全流程。

## 核心功能
1. **自然语言对话交互**：不需要记忆命令，直接用中文口语化对话完成点单
2. **用户偏好记忆**：自动记住用户常用的咖啡类型、冰度、糖度、杯型、配料等偏好
3. **多轮对话支持**：支持上下文理解，支持询问、确认、修改等多轮交互
4. **模拟咖啡店数据**：内置5个主流咖啡品牌的常见菜单数据
5. **完整流程模拟**：模拟订单提交、商家接单、制作、配送全流程状态提醒
6. **历史订单管理**：支持查询历史订单，一键重复下单

## 技术架构
### 模块划分
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  对话交互层     │───▶│  意图识别层     │───▶│  订单处理层     │
│  (REPL模式)     │    │  (NLU)          │    │  (业务逻辑)     │
└─────────────────┘    └─────────────────┘    └─────────────────┘
                                                  │
                                                  ▼
                                        ┌─────────────────┐
                                        │  数据存储层     │
                                        │  (本地JSON)     │
                                        └─────────────────┘
                                                  │
                                                  ▼
                                        ┌─────────────────┐
                                        │  模拟数据层     │
                                        │  (咖啡店菜单)   │
                                        └─────────────────┘
```

### 各模块职责
1. **对话交互层**：
   - 提供REPL命令行交互界面
   - 处理用户输入和系统输出
   - 维护对话上下文

2. **意图识别层**：
   - 识别用户输入的意图类型
   - 提取关键参数（咖啡种类、冰度、糖度、杯型、配料等）
   - 支持的意图类型：
     - 点单意图
     - 查询历史订单意图
     - 重复下单意图
     - 询问推荐意图
     - 确认/取消意图
     - 帮助/问候意图

3. **订单处理层**：
   - 匹配咖啡店和商品
   - 验证订单参数，补全用户偏好
   - 生成订单，维护订单状态
   - 模拟订单状态流转

4. **数据存储层**：
   - 用户偏好数据存储：`data/user_preferences.json`
   - 历史订单数据存储：`data/order_history.json`
   - 咖啡店菜单数据存储：`data/coffee_shops.json`

5. **模拟数据层**：
   - 内置5个咖啡品牌：瑞幸、星巴克、Manner、Tims、Seesaw
   - 每个品牌包含10-15个常见商品
   - 包含商品价格、规格选项等信息

## 交互流程示例
### 正常点单流程
```
用户：帮我点一杯少冰半糖的大杯拿铁，加奶盖
Agent：好的，为您确认订单：
  店铺：瑞幸咖啡（默认偏好）
  商品：大杯拿铁 少冰 半糖 + 奶盖
  价格：24元
  是否确认下单？
用户：确认
Agent：🎉 订单已提交，订单号：CF20260421001
Agent：⏳ 商家已接单，正在制作中...
（等待3秒）
Agent：🍵 咖啡已做好，骑手正在配送，预计15分钟送达~
（等待5秒）
Agent：🛵 您的咖啡已送到，请取餐~
```

### 重复下单流程
```
用户：再给我来一杯和上次一样的
Agent：好的，为您重复上次订单：
  店铺：瑞幸咖啡
  商品：大杯拿铁 少冰 半糖 + 奶盖
  价格：24元
  是否确认下单？
用户：确认
...
```

### 查询历史订单
```
用户：我最近点过什么？
Agent：您最近的3笔订单：
1. 2026-04-21 18:30 瑞幸 大杯拿铁 少冰半糖 +奶盖 24元
2. 2026-04-20 09:15 星巴克 中杯美式 热 无糖 28元
3. 2026-04-19 14:00 Manner 小杯冰美式 15元
```

## 技术实现
- 开发语言：Python 3.10+
- 交互框架：标准库实现REPL模式，不需要额外依赖
- 数据存储：JSON文件，轻量无数据库
- 意图识别：规则匹配实现，简单高效，不需要大模型（后续可扩展接入LLM）
- 状态流转：有限状态机实现订单状态管理
